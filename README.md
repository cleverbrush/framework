# Cleverbrush Framework

This repository contains utility libraries used by [Cleverbrush Online Vector Editor](https://cleverbrush.com).

All of these packages are open-source and free to use.

Libraries are written in **TypeScript** and distributed as **ES6 modules**. You can use them with any bundler or transpiler.

## Packages

| Package | Description |
| --- | --- |
| [`@cleverbrush/async`](./libs/async) | Async utilities: `Collector`, `debounce`, `throttle`, `retry` |
| [`@cleverbrush/deep`](./libs/deep) | Deep operations on objects: equality, merge, flatten |
| [`@cleverbrush/schema`](./libs/schema) | Schema definition and validation with TypeScript type inference |
| [`@cleverbrush/mapper`](./libs/mapper) | Schema-to-schema object mapping with type-safe property selectors |
| [`@cleverbrush/scheduler`](./libs/scheduler) | Job scheduler for Node.js using worker threads |
| [`@cleverbrush/knex-clickhouse`](./libs/knex-clickhouse) | Knex query builder dialect for ClickHouse |

## Development

This project uses [npm workspaces](https://docs.npmjs.com/cli/using-npm/workspaces). All libraries are located under the `libs` directory.

### Build

```bash
npm run build
```

### Test

All libraries are covered with unit tests. We use [Vitest](https://vitest.dev/) as a test runner.

```bash
npm test
```

### Documentation

API documentation is generated by [TypeDoc](https://typedoc.org/) and published at https://docs.cleverbrush.com/.

```bash
npm run docs
```

Each library also has its own `README.md` with usage examples and API reference.

## Contributing

If you are going to create a pull request, please make sure that your changes are covered with tests and don't break existing ones. If you add a new feature or fix a bug, please update the documentation as well — usually it's just a matter of editing JSDoc comments.
